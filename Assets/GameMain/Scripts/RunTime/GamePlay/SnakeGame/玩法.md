# 贪吃蛇玩法总结

## 1. 核心机制
本玩法是基于经典贪吃蛇的网格化解谜模式。玩家控制蛇头在网格中移动，身体会随着移动而跟随。

## 2. 移动规则
- **自动移动**：蛇以固定时间间隔自动向前移动一格。
- **转向控制**：玩家可以控制上下左右四个方向。
- **成长**: 蛇每移动一格, 都会长长一格
- **限制**：
    - 不能直接进行 180 度调头。
    - 如果侧面紧贴墙壁，无法向该侧转向。

## 3. 收集与成长
- **蜂蜜 (SnakeHoney)**：蛇碰到蜂蜜会收集。
- **临时状态**：蜂蜜在收集后，如果蛇死亡，蜂蜜回到未收集状态。

## 4. 死亡与复活
- **碰撞判定**：撞到墙壁、障碍物或自己的**活动身体**会导致死亡。
- **复活**：死亡后会在上一个检查点（SavePoint）复活，且蛇的长度会重置为0。

## 5. 永久身体 (Permanent Tails)
- **检查点转化**：当蛇头到达检查点时，当前的**所有活动身体**都会转化为“永久身体”。
- **永久身体特性**：
    - 变为半透明。
    - 不再具有碰撞判定，蛇头可以穿过它们。
    - 状态会被保存，即便死亡或重新加载场景也会存在。

## 6. 机关门机制 (SnakeDoor)
- **解锁条件**：每个机关门关联若干个蜂蜜，只有当所有关联蜂蜜都被收集时，门才会打开。
- **永久开启**：
    - 门打开是临时的，如果蛇死亡，门会重新关闭。
    - 玩家必须在**门开启的状态下到达检查点**，才能将该门永久开启。

# 工程要求:

1. 各个物品动画都易于配置
2. 蛇可以多种尾巴变体(每种变体都存在Resources文件夹下, 设置好尾巴路径, 蛇会自己构建一个尾巴库. 尾巴只有两种,直线和直角)
3. 所有脚本都是一键配置的, 给空物体添加脚本后功能就正常了
4. 要能完美的兼容当前游戏架构. 关卡开始时, Player会被通过PlayerSnakeInitializer进行初始化. 其它的entity则注意OnEnable和OnDisable.
5. 遵循项目规范, 可以阅读README.md 和 CLAUDE.md
